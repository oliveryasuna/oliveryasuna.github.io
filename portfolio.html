Hyper

Personal project for financial management. Aggregates and analyzes financial information from multiple sources, including bank, brokerage, 401(k), IRA, 529 Plan, and insurance accounts, credit cards, and loans. Provides utilities to analyzes transactions and trades.

This application utilizes the following concepts, tools, and frameworks:

  • Spring: As the Java EE application framework in general and for RESTful APIs.
  • Hibernate & JPA: For modeling and mapping relational data between a MySQL database.
  • Vaadin 14: For the front-end user-facing interface.
  • Tomcat: As the container for applicable Servlets.
  • Selenium: For browser automation to acquire account information belonging to services without public APIs.
  • JSoup: For use with Selenium to parse DOM where applicable. Selenium validates element handles before performing operations, which is unnecessary in all cases and can be significantly slower compared to static parsing with JSoup.

It is currently under development and the source is not public. I am happy to discuss it further.

Gimme DOM

A Vaadin 8 extension enabling developers to have more control over the web DOM. My goal here is to solve Vaadin 8's core problem: it does not feel like you are developing web applications. For example, some components lack common event handlers. Others lack important DOM event handlers altogether. Vaadin does not enable developers to modify DOM directly. Gimme DOM aims to implement this.
